## Acceptance Criteria & Specs

### SearchBar (ML-01)
- Props: `placeholder?: string` (default: 'Buscar...'), `value?: string` (controlled), `defaultValue?: string` (uncontrolled, default: ''), `onSearch?: (query: string) => void`, `onChange?: (value: string) => void`, `loading?: boolean` (default: false), `disabled?: boolean` (default: false), `showButton?: boolean` (default: true), `buttonLabel?: string` (default: 'Buscar'), `size?: 'sm' | 'md' | 'lg'` (default: 'md').
- Wrapper with `role="search"` and `aria-label="Barra de búsqueda"`.
- Focus, sizing (coordinate Input and Button height), and layout (`display: flex`, `gap: var(--space-2)`).
- Input prefix: search Icon (sm/md/lg based on size).
- Behaviour: Enter on Input or click on Button calls `onSearch(value)`. Escape on Input resets value to empty (if uncontrolled) and triggers `onChange('')`. If `loading=true`, Button is loading (shows spinner), Input is disabled. If `showButton=false`, only Input with icon prefix is displayed.

### FormField (ML-02)
- Props: `label: string` (required), `htmlFor?: string`, `required?: boolean` (default: false), `error?: string`, `hint?: string`, `children: React.ReactElement` (required).
- Render label on top of input field. Asterisk red `*` next to label if `required` is true (`aria-hidden="true"`).
- Render hint under child when there is no error.
- Render error in red under child, replacing hint.
- Injects dynamically into children:
  - `id` (htmlFor or autogenerated via React's `useId()`).
  - `aria-describedby` (pointing to errorId or hintId).
  - `error` prop (pass the error string to trigger children error styling).
- Uses `React.Children.only` and `React.cloneElement`.
